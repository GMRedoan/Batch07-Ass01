## Question: How do the four pillars of OOP—Inheritance, Polymorphism, Abstraction, and Encapsulation—help manage logic and reduce complexity in large-scale TypeScript projects?

# Answer:
In large TypeScript projects, the four pillars of OOP help organize code, reduce duplication, and make systems easier to maintain.

- `Encapsulation`
Bundles data and methods inside a class.
Hides internal details using private and protected.
Prevents unwanted changes from outside code.


- `Abstraction`
Shows only essential features, hides implementation details.
Achieved using interfaces and abstract classes.


- `Inheritance`
Allows reuse of common logic from a parent class.
Child classes extend and customize behavior.


- `Polymorphism`
Same method behaves differently in different classes.
Achieved through interfaces or method overriding.
